Method

How it works

The index scores only what public data can actually support — two axes — and reads every IP by where it lands. It does not produce a single mysterious score or a dollar figure.

1

Source strength

How strong the original is, from four public signals: Reach (views), Engagement (ratings, fandom), Scale (chapters), Momentum (recent trend). 0–100 index.

.40·Reach + .25·Engagement + .20·Scale + .15·Momentum
2

Optionality

How many distinct-audience formats it can credibly carry — drama, film, anime — weighted by each format's reach. Drama, film and anime audiences barely overlap, so breadth spreads risk. 0–100 index.

Σ ( format fit × format reach ) ÷ 2.55
3

Place it

High/low on each axis gives four opportunity types. The type — not a score — is the output: it says how to play the IP.

source × optionality → quadrant

The four types

FRANCHISE
high source · high optionality
Big original that carries 3 formats. Build the whole franchise. Highest ceiling, most competition — usually already optioned.
NICHE MULTI
lower source · high optionality
Smaller original, broad fit. A low-cost way to spread small bets across several audiences.
TENTPOLE
high source · narrow optionality
Big original that fits one format strongly. A single concentrated bet — often anime for action IP.
PASS
lower source · narrow fit
Neither big nor broad. Not worth securing an option now.

Validation & honest limits

Why it holds up — and where it stops

Format fit is judged by an AI from the synopsis alone, no genre label, so it can't echo the answer. On the adapted set it recovers the real format 18 of 20 times — a non-circular check that the optionality axis is measuring something real.

Why payoff is not an axis

An earlier version multiplied in a payoff estimate. In testing, fit predicts payoff at only Spearman ρ ≈ 0.62, and that is almost all floor-separation: low-fit titles reliably flop, but the top-fit picks include none of the three biggest hits. It predicts which format well, but how big badly. So payoff is demoted to a reference band and never drives the classification.

Why no single score, and no dollars

Folding a strong signal (optionality) and a weak one (payoff) into one number produced a score nobody could interpret — so OSMU-as-a-ranking was dropped. And converting to won or dollars would need budgets, licensing fees and revenue splits, all private. A made-up figure would look precise and be wrong. The index stops where the public data does.

Source strength is decomposable

Each source score breaks into Reach, Engagement, Scale and Momentum, so it's inspectable rather than a hand-wave. The momentum term matters: Mount Hua Sect has enormous reach and scale (~700M novel views, 1,900+ chapters) yet its ratings and readership are past peak — so its source score lands well below its raw size. Big is not the same as rising.

What's next

  • Source strength is curated from public research today — a reproducible readership pipeline would replace hand-set tiers.
  • Wider, automated pool — scaling past 24 hand-picked titles needs an automated crawl of platform rankings.
  • Multi-source — extend the source signal consistently across novels, games and original IP.
